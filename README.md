# BioMedicalRAG

## Project Overview

This project studies how different retrieval corpora and retrieval methods affect the quality of medical QA responses generated by open-source LLMs (Qwen2.5-7B, Llama-3.1-8B, Mistral-7B, and their 70B/72B variants).

**Retrieval Corpora:**
- `bioasq` — PubMed abstracts from BioASQ Task A (~16M documents)
- `yahoo` — Yahoo Answers health-related Q&A
- `medical_textbooks` — Medical textbooks from HuggingFace MedRAG
- `healthcaremagic` — HealthCareMagic patient-doctor conversations

**Retrieval Methods:**
- `bm25` — Tantivy-based BM25 sparse retrieval (title-boosted)
- `tfidf` — TF-IDF sparse retrieval (scikit-learn)
- `medcpt` — MedCPT dense retrieval (NCBI biomedical bi-encoder)
- `hybrid` — Reciprocal Rank Fusion (RRF) of BM25 + MedCPT

**Query Datasets:** MeQSum, BioASQ Task B, MedRedQA, MedQuAD, MedicationQA, MedQA (USMLE), MASH-QA, MedMCQA, ChatDoctor-iCliniq, MMLU Medical

**Generator Models:** Qwen2.5-7B/72B-Instruct, Llama-3.1-8B/70B-Instruct, Mistral-7B-Instruct-v0.3

**Evaluation Metrics:** ROUGE-1/2/L, BLEU, METEOR, BERTScore, MCQ accuracy

## Setup

### 1. Install Dependencies

```bash
pip install -r requirements.txt
```

**Optional — vLLM (for 70B/72B models):**
```bash
pip install vllm
```

### 2. Configure Paths

Edit `config.py` to set your HuggingFace cache directory:
```python
os.environ["HF_HOME"] = "/your/path/to/hugging_face"
HF_HOME = "/your/path/to/hugging_face"
```

Optionally create a `.env` file in the project root:
```
HF_TOKEN=hf_your_token_here
```

### 3. Set HuggingFace Token

Export your HuggingFace token (required for gated models like Llama):
```bash
export HF_TOKEN=hf_your_token_here
```

## Running the Main Experiment

The main experiment runs BM25 retrieval with BioASQ and Yahoo corpora across multiple query datasets. All steps can be run individually or all at once.

### Full Pipeline (All Steps)

```bash
python main.py --step all --generator-model qwen2.5-7b
```

### Step-by-Step

**Step 1: Build Corpora**
```bash
python main.py --step build_corpora --generator-model qwen2.5-7b
```
Downloads and processes BioASQ, Yahoo Answers, Medical Textbooks, and HealthCareMagic corpora into JSONL format under `data/processed/`.

**Step 2: Build BM25 Indexes**
```bash
python main.py --step build_indexes --corpus bioasq yahoo medical_textbooks healthcaremagic
```
Builds on-disk Tantivy BM25 indexes under `indexes/`.

**Step 3: Load Datasets**
```bash
python main.py --step load_datasets --generator-model qwen2.5-7b
```
Downloads and preprocesses all 10 query datasets into `data/processed/`.

**Step 4: Run Experiments**
```bash
python main.py --step run_experiments \
  --generator-model qwen2.5-7b \
  --conditions A B C D E F
```
Available condition families:
- `A, B` — Baseline (no RAG), MeQSum (lay) / BioASQ Task B (expert)
- `C, D` — RAG-BioASQ, MeQSum / BioASQ Task B
- `E, F` — RAG-Yahoo, MeQSum / BioASQ Task B
- `A1–F1` — Same but with MedRedQA / MedQuAD
- `A2–F2` — MedicationQA / MedQA
- `A3–F3` — MASH-QA / MedMCQA
- `A4–F4` — ChatDoctor-iCliniq / MMLU Medical

**Step 5: Merge Results**
```bash
python main.py --step merge_results --generator-model qwen2.5-7b
```

**Step 6: Evaluate**
```bash
python main.py --step evaluate --generator-model qwen2.5-7b
```
Computes ROUGE, BLEU, METEOR, BERTScore, MCQ accuracy, readability, and jargon density. Outputs to `results/<model>/evaluated_results.csv`.

### Running on Multiple Models

```bash
for MODEL in qwen2.5-7b llama3.1-8b mistral-7b; do
  python main.py --step run_experiments --generator-model $MODEL --conditions A B C D E F
  python main.py --step merge_results --generator-model $MODEL
  python main.py --step evaluate --generator-model $MODEL
done
```

### Using GPU Selection

```bash
CUDA_VISIBLE_DEVICES=0 python main.py --step run_experiments --generator-model qwen2.5-7b
```

## Extended Experiment (All Retrievers)

The extended pipeline adds TF-IDF, MedCPT dense, and Hybrid (RRF) retrieval on top of BM25.

### Full Extended Pipeline

```bash
python run_extended.py --step all --generator-model qwen2.5-7b
```

### Step-by-Step

**Step 1: Build All Indexes**
```bash
python run_extended.py --step build_indexes
```
Builds BM25 (Tantivy), TF-IDF (pickle), and MedCPT (dense embeddings) indexes.

**Step 2: Pre-compute Retrieval Cache (Recommended)**

Pre-computing retrievals avoids GPU contention between MedCPT and the LLM generator:
```bash
python precompute_retrievals.py
```
Options:
```bash
python precompute_retrievals.py --retriever bm25
python precompute_retrievals.py --retriever hybrid --corpus bioasq
python precompute_retrievals.py --conditions G H I J K L M N O P Q R
python precompute_retrievals.py --save-top-k 50
```

**Step 3: Run Extended Conditions**
```bash
python run_extended.py --step run_experiments \
  --generator-model qwen2.5-7b \
  --conditions G H I J K L M N O P Q R
```

Extended condition families (on BioASQ + Yahoo corpora):
| Conditions | Retriever | Corpus |
|---|---|---|
| A, B | Baseline | None |
| C–F | BM25 | BioASQ / Yahoo |
| G–J | TF-IDF | BioASQ / Yahoo |
| K–N | MedCPT | BioASQ / Yahoo |
| O–R | Hybrid (RRF) | BioASQ / Yahoo |

Additional corpus families (Medical Textbooks + HealthCareMagic):
| Conditions | Retriever | Corpus |
|---|---|---|
| A11–R11 | All retrievers | MedicalTextbooks / HealthCareMagic |
| A22–R22 | All retrievers | MedicalTextbooks + MedicationQA/MedQA |
| A1N–R1N | All retrievers | MedicalTextbooks + MedRedQA/MedQuAD |
| A33–R33 | All retrievers | MedicalTextbooks + MASH-QA/MedMCQA |
| A44–R44 | All retrievers | MedicalTextbooks + ChatDoctor/MMLU |

**Step 4: Merge Results**
```bash
python run_extended.py --step merge_results --generator-model qwen2.5-7b
```

**Step 5: Evaluate**
```bash
python run_extended.py --step evaluate --generator-model qwen2.5-7b
```
Outputs: `results/<model>/evaluated_results_extended.csv` and `condition_summary_extended.csv`.

### vLLM (For 70B/72B Models)

```bash
CUDA_VISIBLE_DEVICES=6,7 python run_extended.py \
  --use-vllm \
  --generator-model qwen2.5-72b \
  --step run_experiments \
  --conditions A B C D E F G H I J K L M N O P Q R
```

Or using the `--cuda-devices` flag:
```bash
python run_extended.py \
  --cuda-devices 6,7 \
  --use-vllm \
  --generator-model qwen2.5-72b \
  --step run_experiments
```

## Ablation Studies

All ablations use a smaller subset from `ablation_subset/<dataset>/` (must be created from the full test sets). The `retrieved_subset_cache/` directory stores precomputed retrievals for fast k-value and few-shot ablations.

### Top-k Retrieval Ablation

Varies the number of retrieved passages (k = 1, 3, 5, 10, 25, 50).

**Run for all conditions and k-values:**
```bash
python ablations/k_values/run_k_values.py \
  --models qwen2.5-7b llama3.1-8b mistral-7b \
  --k-values 1 3 5 10 25 50
```

**Run for specific conditions:**
```bash
python ablations/k_values/run_k_values.py \
  --models qwen2.5-7b \
  --conditions A B C D E F G H I J K L M N O P Q R \
  --k-values 1 3 5
```

**Run with vLLM:**
```bash
python ablations/k_values/run_k_values.py \
  --models qwen2.5-72b \
  --use-vllm \
  --cuda-devices 6,7 \
  --k-values 1 3 5 10 25 50
```

**Only merge results (no generation):**
```bash
python ablations/k_values/run_k_values.py \
  --step merge_results \
  --models qwen2.5-7b \
  --k-values 1 3 5 10 25 50
```

**Outputs:** `results/<model>/ablations/k-values/k_<k>/raw_outputs/`

### Few-Shot Ablation

Varies the number of in-context examples (shots = 1, 3, 5, 10).

**Run for all conditions and shot counts:**
```bash
python ablations/few_shot/run_few_shot.py \
  --models qwen2.5-7b llama3.1-8b mistral-7b \
  --shots 1 3 5 10
```

**Run for specific conditions:**
```bash
python ablations/few_shot/run_few_shot.py \
  --models qwen2.5-7b \
  --conditions A B C D E F \
  --shots 1 3 5 \
  --seed 42
```

**Run with vLLM:**
```bash
python ablations/few_shot/run_few_shot.py \
  --models qwen2.5-72b \
  --use-vllm \
  --cuda-devices 6,7 \
  --shots 1 3 5 10
```

**Only merge results:**
```bash
python ablations/few_shot/run_few_shot.py \
  --step merge_results \
  --models qwen2.5-7b \
  --shots 1 3 5 10
```

**Outputs:** `results/<model>/ablations/few-shot/shots_<n>/seed_<seed>/raw_outputs/`

**Note:** Few-shot ablation requires training pools in `training/<dataset>/<dataset>_training_pool.csv`.

## Configuration

Key parameters in `config.py`:

| Parameter | Default | Description |
|---|---|---|
| `TOP_K` | 5 | Number of retrieved passages |
| `GENERATION_BATCH_SIZE` | 16 | Batch size for HF generator |
| `SEED` | 42 | Random seed |
| `DO_SAMPLE` | False | Greedy decoding by default |

Key parameters in `config_extended.py`:

| Parameter | Default | Description |
|---|---|---|
| `BM25_TITLE_BOOST` | 2 | Title field weight for BM25/TF-IDF |
| `RRF_K` | 60 | RRF constant for hybrid retrieval |
| `MEDCPT_BATCH_SIZE` | 64 | Batch size for MedCPT encoding |
